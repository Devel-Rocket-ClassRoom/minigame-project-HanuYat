# 프로젝트 개요
- 게임 제목: HanuYat 미니게임 (Project HanuYat)
- 핵심 콘셉트: 플레이어가 학교 복도를 이동하는 학교 테마의 게임.
- 플레이어: 싱글 플레이어
- 렌더 파이프라인: URP (PC_RPAsset)
- 대상 플랫폼: Standalone Windows

# 게임 메커니즘
## 복도 레이아웃
- 복도는 고정된 3x20 그리드(총 60개 타일)로 구성됩니다.
- 타일 크기: 2.5 x 2.5 유닛.
- **고정 베이스 맵 패턴 (테두리 강조 스타일)**: 
    - 왼쪽(x=0)과 오른쪽(x=2) 열은 `Floor_02`를 사용합니다.
    - 중앙(x=1) 열은 `Floor_01`을 사용합니다.
    - 스크립트 부착 없이 씬에 게임 오브젝트를 직접 배치하여 고정된 맵을 만듭니다.

# UI
- 해당 없음

# 주요 에셋 및 컨텍스트
- 바닥 프리팹: 현재 `Main Scene`의 `Corridor` 오브젝트 하위에 있는 `Floor_01` 및 `Floor_02`.
- 대상 부모 오브젝트: `Main Scene`의 `Corridor` 게임 오브젝트.

# 구현 단계
1. **타일 배치 실행 (일회성 에디터 스크립트 방식)**
   - 별도의 컴포넌트를 부착하지 않고, 에디터 도구를 사용하여 3x20 그리드에 타일을 직접 생성합니다.
   - 기존에 `Corridor` 하위에 있던 임시 타일(`Floor_01`, `Floor_02`)을 프리팹 원본으로 활용하여 복제합니다.
   - 배치 로직:
     - 좌우 끝 열: `Floor_02` 배치
     - 중앙 열: `Floor_01` 배치
     - 위치 계산: `x * 2.5`, `z * 2.5` 좌표에 정렬

2. **기존 오브젝트 정리**
   - 배치 완료 후, 원본으로 사용했던 `Corridor` 하위의 개별 타일들은 정리하거나 구조에 포함시킵니다.

# 검증 및 테스트
- **시각적 확인**: 3개 열(가장자리-중앙-가장자리) 구조가 20줄(총 60개)로 올바르게 배치되었는지 확인합니다.
- **정렬 확인**: 타일들이 2.5 유닛 간격으로 빈틈없이 붙어 있는지 확인합니다.
- **계층 구조 확인**: 생성된 모든 타일이 `Corridor` 오브젝트의 자식으로 올바르게 위치해 있는지 확인합니다.
